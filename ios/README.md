# OrbitPay iOS — Build Guide

This directory is the iOS wrapper for the OrbitPay Member Portal.
The wrapper loads `https://orbitpaybank.online` inside a native WKWebView,
giving your users a real iOS app with push notifications, biometric
login, Face ID, and offline support.

## One-time setup

### Prerequisites
- macOS 13+ (Ventura) or newer
- Xcode 15+ (download from https://apps.apple.com/us/app/xcode/)
- CocoaPods (`brew install cocoapods`)
- An active Apple Developer account ($99/yr) — required for App Store
  distribution and for signing the .ipa with a real provisioning profile.
  Sign up at https://developer.apple.com/programs/enroll/

### Required secrets (set as environment variables in your shell)

```bash
export APPLE_DEVELOPMENT_TEAM="XXXXXXXXXX"        # 10-char team ID from developer.apple.com
export APP_BUNDLE_ID="com.orbitpay.creditunion"
export APP_VERSION="1.0.0"
export APP_BUILD_NUMBER="1"
export PROVISIONING_PROFILE_NAME="OrbitPay Member App"
```

### Build steps

```bash
# 1. From the project root, build the web bundle
npm install
npm run build

# 2. Initialize the iOS project (one-time, scaffolds Xcode project)
npx cap add ios

# 3. Copy the latest web bundle + plugins into ios/
npx cap sync ios

# 4. Open in Xcode to set signing + bundle ID
npx cap open ios

# 5. In Xcode:
#    - Select the "App" target
#    - Signing & Capabilities → select your Apple Developer team
#    - Build Settings → set PRODUCT_BUNDLE_IDENTIFIER to com.orbitpay.creditunion
#    - Set deployment target to iOS 15.0+
#    - Product → Archive
#    - In the Organizer: Distribute App → App Store Connect → Upload

# 6. Submit for App Store review at https://appstoreconnect.apple.com
```

## Configuration files

- `capacitor.config.ts` — at the project root, controls app metadata,
  server URL, native plugin behavior. Edit there, not in Xcode.
- `Info.plist` — generated by `npx cap sync`. Add privacy strings here.
- `AppDelegate.swift` — generated by Capacitor. Custom URL schemes,
  push notifications, deep links — all configured here.

## Required Info.plist additions (privacy strings)

When you submit to the App Store, Apple's review will require:

| Key | Value | Why |
|-----|-------|-----|
| `NSLocationWhenInUseUsageDescription` | "OrbitPay uses your location to find the nearest branch and protect your account from unauthorized access." | For branch finder + fraud detection |
| `NSCameraUsageDescription` | "OrbitPay needs camera access to scan QR codes for payments and verify your identity." | For QR payments + KYC |
| `NSPhotoLibraryUsageDescription` | "OrbitPay needs photo library access so you can attach documents to your applications." | For loan applications + identity verification |
| `NSFaceIDUsageDescription` | "OrbitPay uses Face ID for fast, secure sign-in." | For biometric login |
| `NSUserTrackingUsageDescription` | "OrbitPay does not track you across other apps." | Required by Apple even if you don't track |
| `NSAppTransportSecurity` | NSAllowsArbitraryLoadsWebContent = NO | Forces HTTPS |

Add these via Xcode → Info.plist or by editing the file after `cap add ios`.

## Push notifications

1. Log in to https://developer.apple.com → Certificates, Identifiers & Profiles
2. Create an APNs Auth Key (.p8 file)
3. Add it to Supabase Dashboard → Settings → API → Push Notifications
4. Use `PushNotifications` Capacitor plugin from `@capacitor/push-notifications`
5. Register device tokens from the client app, store in Supabase `devices` table

## Building without Xcode (CI)

You can build the .ipa on a CI machine using `xcodebuild` directly:

```bash
xcodebuild \
  -workspace ios/App/App.xcworkspace \
  -scheme App \
  -configuration Release \
  -archivePath build/OrbitPay.xcarchive \
  archive

xcodebuild \
  -exportArchive \
  -archivePath build/OrbitPay.xcarchive \
  -exportPath build/ipa \
  -exportOptionsPlist ios/App/ExportOptions.plist
```

The result is `build/ipa/OrbitPay.ipa`, which you can upload to App Store
Connect via `xcrun altool --upload-app -f build/ipa/OrbitPay.ipa -u your@apple.id`.

## Submission to App Store

1. Apple App Store Connect: https://appstoreconnect.apple.com
2. My Apps → "+" → New App → iOS
3. Bundle ID: `com.orbitpay.creditunion`
4. SKU: `orbitpay-member-app`
5. Fill in screenshots (required sizes listed in App Store Connect),
   description, keywords, support URL, marketing URL
6. Upload the build via Xcode Organizer or `xcrun altool`
7. Submit for review

## Review timeline

Apple typically reviews within 24-48 hours. Rejections are common
on first submission — the most frequent issues:
- Missing privacy strings in Info.plist
- App references web content without sufficient native functionality
  (Apple wants the app to do *something* native, not just wrap a website)
- Placeholder content

To strengthen the case for "real" native functionality, OrbitPay's
app could add:
- Touch ID / Face ID sign-in via the @capacitor/biometric plugin
- Push notifications for transactions
- Apple Pay integration
- Offline transaction history cache
- Native share sheet for referral links

## Internal testing (TestFlight)

Before public release, distribute to internal testers via TestFlight:

1. App Store Connect → My Apps → OrbitPay → TestFlight
2. Upload the .ipa (Xcode Organizer → Distribute → TestFlight)
3. Add internal testers by email
4. Testers install the TestFlight app and accept the invite

## Updates

After your app is live, every push to `main` should:
1. Run `npm run build && npx cap sync ios` to refresh the bundle
2. Increment `APP_BUILD_NUMBER` in your CI
3. Re-archive and re-upload to App Store Connect
4. Submit for review (Apple approves updates faster than first submissions)

## Why this approach (Capacitor, not React Native)

- **Same codebase as the website** — one React app, three deployment targets
  (web, iOS, Android). No duplicated business logic.
- **Real native features** — biometrics, push notifications, Apple Pay all
  available via plugins.
- **App Store compliant** — the wrapper is more than a thin shell because
  the OrbitPay portal is a full-featured financial app, not a marketing site.
- **Easier maintenance** — every feature you ship on the web ships on
  mobile the same day, no separate iOS team needed.
